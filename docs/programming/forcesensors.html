Force Sensors

Rigid Body Force Sensors

Force sensors can be attached to rigid bodies to measure forces and torques experienced at user-specified reference frames. The readings represent the net forces and torques experienced by the parent body, which include external forces, contact forces, and internal forces applied by the solver (e.g., due to joint drives). A body resting on the ground plane will have a net force of zero.(力传感器可以附加到刚体上，以测量在用户指定的参考系下所承受的力和扭矩。读数表示父刚体所承受的净力和净扭矩，包括外力、接触力以及求解器施加的内力（例如，由于关节驱动）。静止在地面上的刚体净力为零。)

Force sensors are created on assets. This way, you only need to define the force sensors once and they will be created on every actor instanced from the asset.(力传感器是在资源上创建的。这样，您只需定义一次力传感器，它们就会在从该资源实例化的每个角色上创建。)

To create a force sensor, you specify the rigid body index to which the sensor will be attached and the relative pose of the sensor with respect to the body origin. Multiple sensors can be attached to the same body:(要创建力传感器，您需要指定传感器要附加到的刚体索引以及传感器相对于刚体原点的相对姿态。多个传感器可以附加到同一个刚体上：)

body_idx = gym.find_asset_rigid_body_index(asset, "bodyName")

sensor_pose1 = gymapi.Transform(gymapi.Vec3(0.2, 0.0, 0.0))
sensor_pose2 = gymapi.Transform(gymapi.Vec3(-0.2, 0.0, 0.0))

sensor_idx1 = gym.create_asset_force_sensor(asset, body_idx, sensor_pose1)
sensor_idx2 = gym.create_asset_force_sensor(asset, body_idx, sensor_pose2)

You can also pass additional properties for each sensor, which determine how the forces will be computed:(您还可以为每个传感器传递附加属性，这些属性决定了力的计算方式：)

sensor_props = gymapi.ForceSensorProperties()
sensor_props.enable_forward_dynamics_forces = True
sensor_props.enable_constraint_solver_forces = True
sensor_props.use_world_frame = False

sensor_idx = gym.create_asset_force_sensor(asset, body_idx, sensor_pose, sensor_props)

See isaacgym.gymapi.ForceSensorProperties for more details.(更多详情请参见 isaacgym.gymapi.ForceSensorProperties。)

After creating actors, you can get the number of attached force sensors and access the individual force sensors using their index:(创建角色后，您可以获取附加的力传感器数量，并使用其索引访问各个力传感器：)

actor_handle = gym.create_actor(env, asset, ...)

num_sensors = gym.get_actor_force_sensor_count(env, actor_handle)
for i in range(num_sensors):
    sensor = gym.get_actor_force_sensor(env, actor_handle, i)

During simulation, you can query the latest sensor readings like this:(在仿真期间，您可以像这样查询最新的传感器读数：)

sensor_data = sensor.get_forces()
print(sensor_data.force)   # force as Vec3
print(sensor_data.torque)  # torque as Vec3

Two sensors attached to the same body will report the same force, but will generally differ in torques if their relative poses are different. Forces are measured at the body's center of mass, but torques are measured at the sensor's local reference frame.(附加到同一刚体上的两个传感器将报告相同的力，但如果它们的相对姿态不同，则扭矩通常会不同。力在刚体的质心处测量，而扭矩在传感器的局部参考系中测量。)

The total number of force sensors in a simulation can be obtained by calling gym.get_sim_force_sensor_count(sim).(仿真中力传感器的总数可以通过调用 gym.get_sim_force_sensor_count(sim) 获得。)

Tensor API

The function acquire_force_sensor_tensor returns a Gym tensor descriptor, which can be wrapped as a PyTorch tensor as discussed in the Tensor API documentation:(函数 acquire_force_sensor_tensor 返回一个 Gym 张量描述符，可以按照 张量 API 文档 中讨论的方式包装为 PyTorch 张量：)

_fsdata = gym.acquire_force_sensor_tensor(sim)
fsdata = gymtorch.wrap_tensor(_fsdata)

The shape of this tensor is (num_force_sensors, 6) and the data type is float32. For each sensor, the first three floats are the force and the last three floats are the torque.(该张量的形状为 (num_force_sensors, 6)，数据类型为 float32。对于每个传感器，前三个浮点数是力，后三个浮点数是扭矩。)

After each simulation step, you can get the latest sensor readings by calling:(在每个仿真步骤之后，您可以通过调用以下方法获取最新的传感器读数：)

gym.refresh_force_sensor_tensor(sim)

Limitations

Force sensors are currently only available with the PhysX backend.(力传感器目前仅在 PhysX 后端可用。)

Joint Force Sensors

To enable reading forces on each degree-of-freedom of articulated actors, call the enable_actor_dof_force_sensors method when setting up the envs:(要启用对关节式角色每个自由度上的力的读取，请在设置环境时调用 enable_actor_dof_force_sensors 方法：)

actor = gym.create_actor(env, ...)
gym.enable_actor_dof_force_sensors(env, actor)

These sensors are not enabled by default for performance reasons. Although the performance impact is usually quite small, it is best to only enable the sensors when needed. During simulation, the forces can be retrieved using the get_actor_dof_forces function:(出于性能原因，这些传感器默认未启用。尽管性能影响通常很小，但最好仅在需要时启用传感器。在仿真期间，可以使用 get_actor_dof_forces 函数获取力：)

forces = gym.get_actor_dof_forces(env, actor_handle)

This returns a numpy array of total forces acting on this actor's DOFs. Note that the returned forces will always be zero if enable_actor_dof_force_sensors was not previously called on the actor.(这将返回一个 numpy 数组，包含作用在此角色 DOF 上的总力。请注意，如果之前未对该角色调用 enable_actor_dof_force_sensors，则返回的力始终为零。)

Tensor API

The function acquire_dof_force_tensor returns a Gym tensor descriptor, which can be wrapped as a PyTorch tensor as discussed in the Tensor API documentation:(函数 acquire_dof_force_tensor 返回一个 Gym 张量描述符，可以按照 张量 API 文档 中讨论的方式包装为 PyTorch 张量：)

_forces = gym.acquire_dof_force_tensor(sim)
forces = gymtorch.wrap_tensor(_forces)

This is a one-dimensional tensor of float32 values corresponding to each DOF in the simulation.(这是一个一维张量，包含 float32 值，对应于仿真中的每个 DOF。)

After each simulation step, you can get the latest sensor readings by calling:(在每个仿真步骤之后，您可以通过调用以下方法获取最新的传感器读数：)

gym.refresh_dof_force_tensor(sim)

Limitations

DOF force sensors are only supported in the PhysX backend.(DOF 力传感器仅在 PhysX 后端中受支持。)
